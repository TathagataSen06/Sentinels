PRODUCTION DEPLOYMENT STEPS:

1. Update .env with production values:
   cp infra/.env.example infra/.env
   # Edit password, log levels, etc.

2. Build images:
   docker compose build

3. Push to registry:
   docker tag sentinels-api:latest myregistry/sentinels-api:v1.0
   docker push myregistry/sentinels-api:v1.0

4. Deploy stack:
   docker compose up -d

5. Monitor status:
   docker compose ps
   docker compose logs

6. Scale if needed:
   docker compose up -d --scale processing-engine=3

==============================================

FOR KUBERNETES DEPLOYMENT:

The same images work with Kubernetes. Example deployment:

  apiVersion: apps/v1
  kind: Deployment
  metadata:
    name: sentinels-api
  spec:
    replicas: 3
    template:
      spec:
        containers:
        - name: api-server
          image: myregistry/sentinels-api:v1.0
          ports:
          - containerPort: 8000
          resources:
            limits:
              cpu: 2
              memory: 1Gi
            requests:
              cpu: 1
              memory: 512Mi

Deploy with: kubectl apply -f deployment.yaml

TROUBLESHOOTING:

Container Won't Start:
  docker compose logs api-server
  docker compose ps

Port Already in Use:
  Edit infra/docker-compose.yml and change ports

Check Disk Space:
  docker system df
  docker system prune -a

View Real-time Stats:
  docker stats

Backup Database:
  docker compose exec postgres pg_dump -U sentinels sentinels > backup.sql
